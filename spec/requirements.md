# 要件定義書 (Requirements Definition)

## 1. プロジェクト概要
音声で操作し、Web API（Qiita記事検索など）を実行するプロトタイプアプリ。
PWA (Progressive Web App) として実装し、iOSのSiriからの呼び出しに対応する。

## 2. ユーザー体験 (UX) フロー
1.  ユーザーが「Hey Siri, Voice Agentを開いて」と発話（またはショートカットを実行）。
2.  Siri（ショートカット）が「何を調べますか？」と音声入力を促す。
3.  ユーザーが「Reactの最新記事」のように発話。
4.  Siriが音声をテキスト化し、PWAのURLをクエリパラメータ付きで開く（例: `https://.../?q=Reactの最新記事`）。
5.  PWAが起動し、自動的にQiita APIを検索。
6.  検索結果（記事タイトル一覧など）を画面に表示。
7.  (オプション) アプリが検索結果を音声合成で読み上げる。

## 3. 機能要件
- **PWA対応**:
    - ホーム画面に追加可能であること（manifest.json）。
    - オフラインでも最低限の画面が表示されること（Service Worker）。
- **Siri連携**:
    - iOSの「ショートカット」アプリ経由で起動できること。
    - 音声入力を受け取り、アプリに渡せること。
- **Qiita検索機能**:
    - Qiita API v2 (`GET /api/v2/items`) を使用。
    - キーワード検索ができること。
- **表示機能**:
    - 検索結果のリスト表示（タイトル、ユーザー、LGTM数）。
    - 記事へのリンク。

## 4. 非機能要件
- **パフォーマンス**: 高速な起動（Vite使用）。
- **制約事項**:
    - iOSのPWA（Standaloneモード）では `Web Speech API` (音声認識) が動作しない制限があるため、**入力はSiriショートカット側で処理する** アーキテクチャとする。
